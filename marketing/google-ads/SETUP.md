# 구글 Ads 셋팅 가이드 (단계별)

> 이 가이드대로 따라하면 `marketing/google-ads/`의 파일들을 구글 Ads 계정에 일괄 업로드해서 광고를 시작할 수 있습니다.

---

## 사전 준비 (한 번만)

### 1. 구글 Ads 계정 생성
- https://ads.google.com → "지금 시작하기"
- 구글 계정으로 로그인 (Google Analytics와 같은 계정 권장 — 연동 편함)
- 비즈니스 정보 입력 후 "전문가 모드로 전환" 클릭 (스마트 모드는 자동화 너무 많아서 비추)

### 2. 결제 정보 등록
- 좌측 하단 ⚙️ → **결제** → 카드 등록
- 결제 방법: 자동 결제 (예: 15만원 청구 후 자동 결제)
- 첫 결제는 ₩500 인증

### 3. Google Analytics 4 연동 (전환 추적 필수)
- ⚙️ → **연결된 계정** → Google Analytics → "연결"
- GA4 속성 "다올리페어 홈페이지" 선택 → 가져오기
- GA4에서 만든 전환 이벤트(예약 폼 제출 등)가 자동으로 구글 Ads에 동기화됨
- **이거 안 해두면 어떤 키워드가 돈 벌어주는지 모름** — 광고 효율 측정 불가

### 4. Google Ads Editor 다운로드
- https://ads.google.com/intl/ko_kr/home/tools/ads-editor/
- 맥용 다운로드 → 설치 → 구글 계정 로그인
- (웹 UI보다 빠르고 일괄 업로드 가능)

## 캠페인 일괄 import (Google Ads Editor)

### 1. 빈 캠페인 구조 만들기 (Editor에서)

Editor 좌측 **+ 만들기** → 캠페인 5개 생성:

| 캠페인명 | 캠페인 유형 | 일일 예산 (시작) | 입찰 전략 |
|---|---|---|---|
| `DR-iPhone` | 검색 | ₩30,000 | 클릭수 최대화 (수동 CPC 전환 후) |
| `DR-iPad` | 검색 | ₩5,000 | 클릭수 최대화 |
| `DR-AppleWatch` | 검색 | ₩5,000 | 클릭수 최대화 |
| `DR-ApplePencil` | 검색 | ₩2,000 | 클릭수 최대화 |
| `DR-MacBook` | 검색 | ₩5,000 | 클릭수 최대화 |

**총 일일 예산: ₩47,000 (월 ~140만원)** — 시작 단계
- 2주 후 데이터 보고 효율 좋은 캠페인 예산 ↑
- 효율 낮으면 ↓

캠페인 설정 공통:
- **위치 타겟팅**: 한국 (또는 수도권으로 좁히기)
- **언어**: 한국어
- **네트워크**: 검색 네트워크만 (디스플레이 OFF — 낭비)
- **광고 일정**: 24시간 (또는 매장 운영 시간 09-21시)
- **기기**: 모바일 입찰 +20% (한국 모바일 검색 비중 ↑)

### 2. 광고그룹 만들기 (각 캠페인 안에)

각 캠페인 안에 4개 광고그룹:
- `General` (전국·일반 키워드)
- `Gasan` (가산점 지역)
- `Sillim` (신림점 지역)
- `Mokdong` (목동점 지역)

### 3. 키워드 일괄 import

Editor에서:
1. 좌측 **키워드 및 타겟팅** → 키워드 → **가져오기** → "파일 선택"
2. `marketing/google-ads/keywords.csv` 선택
3. 미리보기 확인 → "변경사항 적용"
4. 1,708개 키워드가 캠페인·광고그룹·매치 타입과 함께 import 됨

⚠️ Editor가 "기존 캠페인 매칭 안 됨" 에러 내면 → 캠페인명 정확히 일치하는지 확인 (`DR-iPhone` 등)

### 4. 광고 일괄 import

1. **광고** → **가져오기** → 파일 선택
2. `marketing/google-ads/ads.csv` 선택
3. RSA(반응형 검색 광고)가 20개 광고그룹에 자동 배치됨

### 5. 부정 키워드 추가

1. 캠페인 선택 → **부정 키워드** → "추가"
2. `marketing/google-ads/negatives.txt` 파일 열어서 내용 복사
3. 붙여넣기 → "캠페인 수준"으로 추가 (모든 캠페인에 동일 적용)

### 6. 변경사항 게시

Editor 상단 **변경사항 게시** 클릭 → 5개 캠페인 + 20개 광고그룹 + 1,700+ 키워드 + 20개 광고 한 번에 업로드.

## 발사 전 체크리스트

### ✅ 결제 정보 등록 확인
- 등록 안 되어 있으면 광고 안 나감

### ✅ GA4 연동 + 전환 가져오기 확인
- Tools 메뉴 → 전환 → "예약 폼 제출" 등 GA4 이벤트가 보여야 함

### ✅ 사이트 링크 추가 (Ad Extension)
- 캠페인 → 자산 → 사이트 링크 → 추가
- 예시:
  - "수리 일지" → `/articles/`
  - "예약 폼" → `/#booking-form`
  - "후기 1000+" → 카카오맵 후기 URL
  - "리퍼 vs 수리" → `/articles/iphone-refurbish-guide.html`

### ✅ 전화번호 자산 추가
- 캠페인 → 자산 → 전화번호
- 가산점 02-XXX-XXXX (지점별 시간대 분기 가능)

### ✅ 위치 자산 추가
- 캠페인 → 자산 → 위치 → Google Business Profile 연결
- 광고에 매장 주소·전화번호 자동 표시

### ✅ 광고 사전 검토 통과 확인
- 게시 후 1~24시간 검토. 일부 거부될 수 있음 (예: "정직한" 같은 비교 표현)
- 거부 사유 보고 수정

## 첫 1주 모니터링

매일 확인할 것:

1. **노출 수 (Impressions)** — 키워드가 검색되고 있는지
2. **클릭수** — 광고가 매력적인지
3. **CTR (클릭률)** — 5% 이상 좋음 / 2% 미만 광고 카피 개선
4. **CPC (평균 비용)** — 예측보다 너무 비싸면 입찰가 ↓
5. **전환** — 예약 폼 제출 / 전화 클릭 (GA4 연동돼있어야 보임)

### 자동 차단/일시중지 권장 사례
- CTR 1% 미만 키워드 → 매치 타입 Exact만 남기고 Phrase 차단
- CPC가 입찰가의 80% 이상 → 입찰가 ↓
- 노출 많은데 클릭 0 → 광고 카피 재작성

## 2주 후 최적화

### A. 효율 좋은 키워드는 입찰가 ↑
- "전환 1건 이상 + CPC < 2,000원" 키워드 → +50% 입찰

### B. 낭비 키워드 제거
- "노출 100+ / 클릭 0" 또는 "클릭 5+ / 전환 0" → 일시중지

### C. 부정 키워드 추가
- "검색 용어" 보고서 → 의도와 안 맞는 검색어 확인 → 부정 키워드 추가
- 예: "아이폰 수리 방법 동영상" → "동영상" 부정 추가

### D. 광고 카피 A/B 테스트
- RSA 헤드라인 중 "표시 빈도 낮음" 항목 → 새 헤드라인으로 교체

## FAQ

### Q. 네이버 광고도 계속해야 하나?
A. 네. 한국은 검색 점유율 네이버 35% + 구글 60%대로 둘 다 큼. 네이버는 이미 잘 돌아가니 유지하고, 구글로 미발견 트래픽 추가 흡수.

### Q. 광고 시작 후 며칠 만에 효과 나나?
A. 검토 통과 후 1~2일 안에 노출 시작. 클릭은 즉시, 전환은 1~2주 데이터 쌓여야 분석 가능.

### Q. 클릭만 많고 예약 안 들어오면?
A. 랜딩 페이지 문제. 키워드 의도와 랜딩이 안 맞는 경우가 많음. 예: "아이폰 13 액정 수리비" → 가격표 있는 페이지로 보내야지 일반 홈으로 보내면 이탈.

### Q. 입찰가는 어떻게 정함?
A. `_gen.py`가 네이버 입찰가 × 1.7로 변환한 상태. 너무 비싸면 -20% / 노출 너무 적으면 +30%. 시장 보면서 조정.
